## Итерация 5: Веб-интерфейс 📋

### Цель

Единый frontend: обзор состояния, тренды, запись на консультацию.

### Ценность

Диабетик видит динамику целиком; доктор — опциональный доступ к данным пациента.

### Критерии завершения

- [ ] интерфейс диабетика: история, снимки прогресса, тренды
- [ ] данные из того же backend, что и bot
- [ ] базовая запись на онлайн / офлайн консультацию
- [ ] интерфейс доктора (опционально): обзор пациента

### Артефакты

- `web/` — frontend-проект
- интеграция с backend API
